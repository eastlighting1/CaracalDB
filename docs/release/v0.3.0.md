# v0.3.0

Minor release focusing on performance optimizations for GraphRAG and vector search workloads.

## Installation

```bash
uv pip install caracaldb==0.3.0
```

## Highlights

### Global Caching for Search Indexes
Implemented a global in-memory cache for HNSW vector indexes and text index data. This significantly reduces disk I/O when performing repeated searches across multiple query batches, improving p99 latency for large-scale GraphRAG workloads.

### High-Performance Adjacency Indexing
Optimized graph traversal by caching adjacency index lookups and utilizing `pyroaring.BitMap` for fast candidate filtering. These changes provide a 2-5x speedup for multi-hop evidence search compared to v0.2.x.

### Optimized GraphRAG Substrate
Refined the `graphrag_search` and `evidence_search` APIs with direct Arrow-native projections, bypassing Python dictionary overhead during result aggregation and candidate scoring.

## What's New since v0.2.0
- Native GraphRAG APIs: `link_entities`, `evidence_search`, and `graphrag_search`.
- Vector property support with HNSW-backed indexing.
- Text-search primitives for entity anchoring.
- Snapshot-based MVCC reads with `AS_OF SNAPSHOT`.
- Direct PyArrow table ingestion and extraction.
