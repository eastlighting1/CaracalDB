# CaracalDB v0.2.4 Release Notes

**Tag:** `v0.2.4`
**Date:** 2026-05-03
**Distribution:** `dist/caracaldb-0.2.4-py3-none-any.whl` + `dist/caracaldb-0.2.4.tar.gz`

## Highlights

- Added named snapshot management through the public database API:
  `create_snapshot`, `list_snapshots`, and `release_snapshot`.
- Added Tuft `AS_OF SNAPSHOT 'name'` reads. Node and edge scans now filter row
  visibility against the snapshot LSN so data inserted after the snapshot is
  hidden from the query result.
- Added tests for successful snapshot reads, missing snapshot diagnostics, and
  post-snapshot node and edge filtering.
- Updated the snapshot guide with the supported workflow and current
  schema-level caveat for `AS_OF` reads.
- Streamlined node and edge batch selection during storage scans.

## Compatibility

This is a patch release. Existing 0.2.x resource ingest, `.crcl` viewer,
generated diagnostics, `connect`, `define_class`, `insert_nodes`, typed table
ingest, and focused `db.sql()` query APIs remain available.
