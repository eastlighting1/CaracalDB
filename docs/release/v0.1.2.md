# CaracalDB v0.1.2 Release Notes

**Tag:** `v0.1.2`
**Date:** 2026-04-29
**Distribution:** `dist/caracaldb-0.1.2-py3-none-any.whl` + `dist/caracaldb-0.1.2.tar.gz`

## Highlights

- Quickstart examples now use CaracalDB-facing APIs instead of PyArrow and internal node-store wiring.
- Added small convenience methods for first-use workflows: `define_class`, `insert_nodes`, `sql`, `exec`, and `rows`.
- Release and benchmark workflows use a wider benchmark tolerance to avoid blocking patch releases on noisy runners.
- Packed `.crcl` remains the default database format for new `connect()` calls.

## Build verification

```powershell
uv run --with hatch hatch build
# dist/caracaldb-0.1.2.tar.gz
# dist/caracaldb-0.1.2-py3-none-any.whl
```

## Smoke install

```powershell
uv pip install dist/caracaldb-0.1.2-py3-none-any.whl
python -c "import caracaldb as cdb; print(cdb.__version__)"  # 0.1.2
caracal --help
```

## Compatibility

This is a patch release. Existing 0.1.x query, packed database, and storage APIs remain available.
