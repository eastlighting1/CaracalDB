# CaracalDB v0.2.8 Release Notes

**Tag:** `v0.2.8`
**Date:** 2026-05-08
**Distribution:** `dist/caracaldb-0.2.8-py3-none-any.whl` + `dist/caracaldb-0.2.8.tar.gz`

## Highlights

- Added generic node text lookup APIs:
  `create_text_index`, `list_text_indexes`, and `text_search`.
- Persisted text-index data inside the local `.crcl` bundle and rebuild stale
  text indexes after node table changes.
- Materialized property-index lookup data and wired indexed equality
  predicates into Tuft `profile` and `explain` via `indexes_used`.
- Extended `paths` with multi-seed evidence expansion using `sources`,
  `target_node_types`, weighted `path_score`, per-seed caps, and selected
  target properties.
- Added deterministic semantic-neighbor upsert keys when callers omit
  `edge_id`, so repeated semantic-neighborhood ingestion does not duplicate
  edges.
- Added deterministic HNSW rebuild seeding through `random_seed`.
- Added `caracal bench graph_ecosystem`, which reports:
  `semantic_entry_mode=caracal_hnsw`,
  `semantic_reentry_mode=native_result_nodes`,
  `relation_expand_mode=neighbors_api`, and `fallback_flags=[]`.

## Compatibility

This is a patch release. Existing 0.2.x Arrow table ingest, resource ingest,
snapshot reads, `.crcl` viewer, vector search, traversal APIs, Tuft pattern
queries, and profile/explain surfaces remain available.

CaracalDB still does not bundle GraphRAG document loading, extraction,
embedding calls, answer generation, citation policy, or reranking policy.
Those remain adapter responsibilities on top of the reusable graph ecosystem
primitives.
