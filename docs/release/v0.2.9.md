# CaracalDB v0.2.9 Release Notes

**Tag:** `v0.2.9`
**Date:** 2026-05-08
**Distribution:** `dist/caracaldb-0.2.9-py3-none-any.whl` + `dist/caracaldb-0.2.9.tar.gz`

## Highlights

- Added native GraphRAG substrate APIs:
  `link_entities`, graph-boosted `vector_search`, `evidence_search`, and
  `graphrag_search`.
- Added `GraphRAGResult`, which exposes Arrow-backed `entity_links`,
  `semantic_hits`, `evidence_chunks`, `citation_candidates`, `paths`, and a
  machine-readable `profile`.
- Added capability flags for `graphrag.link_entities`,
  `graphrag.evidence_search`, `graphrag.vector_graph_search`,
  `graphrag.search`, `vector_search.graph_boosts`,
  `traversal.evidence_search`, and `arrow.results`.
- Updated `caracal bench graph_ecosystem` to report:
  `semantic_entry_mode=caracal_graphrag_search`,
  `query_entity_linking_mode=caracal_link_entities`,
  `semantic_reentry_mode=native_result_nodes`,
  `relation_expand_mode=caracal_evidence_search`, and `fallback_flags=[]`.

## Compatibility

This is a patch release. Existing 0.2.x Arrow table ingest, resource ingest,
snapshot reads, `.crcl` viewer, vector search, traversal APIs, Tuft pattern
queries, and profile/explain surfaces remain available.

CaracalDB still does not bundle GraphRAG document loading, extraction,
embedding calls, answer generation, citation policy, or reranking policy.
Those remain adapter responsibilities on top of the reusable graph ecosystem
primitives.
