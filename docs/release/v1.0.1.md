# CaracalDB v1.0.1

v1.0.1 is a packaging fix release for the v1.0 line.

## Fixed

- Removed DGL from the default and optional dependency metadata.
- Removed the DGL adapter/backend path from the supported ML adapter surface.
- Kept core installs focused on CaracalDB storage, query, Arrow, vector, and
  Rust extension dependencies.
- Constrained supported Python metadata to `>=3.11,<3.14` so resolvers do not
  attempt unsupported future Python combinations.
- Fixed the release workflow to build one `abi3` wheel per platform, build the
  source distribution with maturin, and verify archives before publishing.
- Exposed `caracaldb.ml.to_graphframe`, `from_graphframe`, `to_pyg_data`, and
  `to_graphs_tuple` as public adapter helpers.

## Install

```bash
uv add caracaldb==1.0.1
```

For direct pip-style installs through uv:

```bash
uv pip install caracaldb==1.0.1
```

Optional ML adapters are still available without DGL:

```bash
uv add "caracaldb[ml]==1.0.1"
```
