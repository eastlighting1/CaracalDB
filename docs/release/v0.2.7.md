# CaracalDB v0.2.7 Release Notes

**Tag:** `v0.2.7`
**Date:** 2026-05-07
**Distribution:** `dist/caracaldb-0.2.7-py3-none-any.whl` + `dist/caracaldb-0.2.7.tar.gz`

## Highlights

- Added graph-addressable vector search through `create_vector_index`,
  `list_vector_indexes`, `drop_vector_index`, `rebuild_vector_index`, and
  `vector_search`.
- Persisted vector-index metadata and HNSW files inside the local `.crcl`
  bundle without adding a server process.
- Added vector distance helpers:
  `cosine_similarity`, `cosine_distance`, `dot_product`, and `l2_distance`.
- Added idempotent Arrow batch upserts for node and edge tables:
  `upsert_node_table_arrow` and `upsert_edge_table_arrow`.
- Added typed traversal and path primitives:
  `neighbors`, `k_hop`, `paths`, and `shortest_path`.
- Added weighted edge and path scoring support using explicit aggregation
  modes: `sum`, `average`, `min`, `max`, and `product`.
- Added Tuft `CALL vector.search(...)` as a semantic entry point that returns
  Arrow-native result batches.
- Added Tuft bounded variable-length path execution for patterns such as
  `MATCH p = (a:Entity)-[:RELATED_TO*1..3]->(b:Entity) RETURN p, length(p)`.
- Added capability discovery and machine-readable profile/explain output for
  the new graph ecosystem surfaces.

## Compatibility

This is a patch release. Existing 0.2.x Arrow table ingest, resource ingest,
snapshot reads, `.crcl` viewer, generated diagnostics, `connect`,
`define_class`, row-based table ingest, adjacency helpers, and focused
`db.sql()` query APIs remain available.

CaracalDB still does not bundle GraphRAG document loading, extraction,
embedding calls, answer generation, or prompt policy. Those remain adapter
responsibilities on top of the new reusable graph ecosystem primitives.
