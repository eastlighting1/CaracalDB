# CaracalDB v1.0.3

v1.0.3 is a patch release for native vector-search performance in repeated
GraphRAG retrieval workloads.

## Fixed

- Fixed the vector entry cache so repeated `vector_search` calls reuse the
  materialized Arrow node table and NumPy vector entries within an open
  database handle.
- Avoided per-query reconstruction of vector entries after an HNSW index has
  already been built or loaded, restoring expected native GraphRAG semantic
  retrieval latency.

## Install

```bash
uv add caracaldb==1.0.3
```

For direct pip-style installs through uv:

```bash
uv pip install caracaldb==1.0.3
```

Optional ML adapters remain available:

```bash
uv add "caracaldb[ml]==1.0.3"
```
