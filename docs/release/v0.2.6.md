# CaracalDB v0.2.6 Release Notes

**Tag:** `v0.2.6`
**Date:** 2026-05-06
**Distribution:** `dist/caracaldb-0.2.6-py3-none-any.whl` + `dist/caracaldb-0.2.6.tar.gz`

## Highlights

- Added a fluent node lookup surface:
  `db.nodes("Person").where(name="Tom Hanks").select("node_id", "name")`.
- Added public adjacency traversal helpers backed by lazy CSR/CSC indexes:
  `db.out(...)`, `db.in_(...)`, and `db.degree(...)`.
- Added graph overlap helpers for recommendation-style workloads:
  `db.common_neighbors(...)` and `db.overlap(...)`.
- Added support for resolving traversal inputs from internal ids, `ResourceRef`
  values, or stored `node_id` values.
- Reused cached CSR/CSC readers while detecting stale index files by vertex and
  edge counts before traversal.
- Invalidated derived graph index files automatically after node or edge
  appends through `Database`.
- Documented the database-level traversal API in the graph API reference.

## Compatibility

This is a patch release. Existing 0.2.x Arrow table ingest, resource ingest,
snapshot reads, `.crcl` viewer, generated diagnostics, `connect`,
`define_class`, row-based table ingest, and focused `db.sql()` query APIs
remain available.
